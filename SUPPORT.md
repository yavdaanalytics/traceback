# Support

## How to get help

| Need | Where |
|------|--------|
| Usage questions, ideas, show-and-tell | [GitHub Discussions](https://github.com/yavdaanalytics/traceback/discussions) |
| Bug reports | [Issues](https://github.com/yavdaanalytics/traceback/issues/new?template=bug_report.yml) |
| Feature requests | [Issues](https://github.com/yavdaanalytics/traceback/issues/new?template=feature_request.yml) |
| Security vulnerability | [`SECURITY.md`](SECURITY.md) (private reporting) |
| Privacy / telemetry | [`docs/PRIVACY.md`](docs/PRIVACY.md), [`docs/TELEMETRY.md`](docs/TELEMETRY.md), https://traceback.yavda.com/privacy |
| Install / hooks / doctor | [`SETUP.md`](SETUP.md) |
| MCP tools | [`docs/API.md`](docs/API.md) |

## Before opening an issue

1. Search existing issues and discussions.
2. Confirm you are on a supported Node version (`>= 22.5.0`).
3. For install problems, include the output of `traceback-setup` / doctor steps from [`SETUP.md`](SETUP.md).

## Maintainer response

This is a small open-source project. We aim to triage new issues within a week when possible, but there is no SLA. Security reports follow the timeline in [`SECURITY.md`](SECURITY.md).

## Commercial / private support

For organization-specific help, contact **Yavda Analytics** via https://traceback.yavda.com.
